## 1. 这次重构想达成什么

把今天分散在 4 个机制里的能力，收敛到 **`using` 块 + 泛型类型 + 签名化 metadata** 三根支柱上：

| 今天的形态 | 目标形态 | 动机 |
| --- | --- | --- |
| `%unique T` / `%shared T` / `%weak T` 等 `%`-前缀类型限定符 | 内建泛型 `Unique<T>` / `Shared<T>` / `Weak<T>` | 所有权种类变成普通类型，可组合、可推断、可作为泛型实参传播 |
| 语句级 `%using(expr) { ... }` 仅作 lifetime fence | `using` 成为统一的"作用域 + 确定性析构 + 条件守卫"原语 | 一个关键字承担所有"进入一个受控作用域"的语义 |
| `%import("foo")` 立即解析、无运行时存在性保证 | `using (var p = %import("foo")) { ... }` 按需加载 + 签名校验后才进入块 | 插件类型的"局部类型安全"——只有签名匹配才让 block 内代码可见该类型 |
| `.zro` 有 typed metadata，但**无全局 token**、跨模块靠 name 字符串 | C# 风格 metadata token 表（TypeDef/MemberRef/AssemblyRef + 签名 blob） | DLL 之间通过**签名**而非物理布局访问；为插件守卫提供稳定身份 |
| `enum`（仅常量值，不带数据） | 新增 `union`（Rust enum 风格，variant 可带字段），用 `switch` / `using` 分支 | 带状态的代数数据类型；与所有权/作用域天然组合 |

## 3. 贯穿全局的设计原则

- **零 GC 写法优先**：owner 路径（`Unique`/`Shared`/`Weak`）走确定性析构与引用计数，不进 GC 扫描集合；只有显式 `%detach` / bridge 才回到 plain GC 世界。沿用 [owned-field-lifecycle.md](../../parser-and-semantics/owned-field-lifecycle.md) 既定边界。
- **`%` 收敛**：`%` 前缀长期只保留**编译器指令/保留字**语义（`%module`、`%import`、`%test`、`%async`、`%type`、`%detach`），所有权"种类"全部迁出 `%`、改用泛型类型。`%unique T` 在过渡期保留为 `Unique<T>` 的语法糖。
- **签名即契约**：跨模块（尤其跨 DLL 插件）一律按 metadata 签名 token 解析，不依赖编译顺序或物理 slot 布局。
- **作用域即生命周期**：`using` 块的词法边界就是资源的生命周期边界，编译期生成确定性 cleanup plan（已存在的 `SZrDeterministicCleanupStep` 机制扩展即可）。

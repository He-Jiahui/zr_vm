## 1. 核心想法

所有权"种类"今天是挂在 `SZrType.ownershipQualifier` 上的一个 enum 字段。把它提升为**内建泛型类型**，让所有权进入正常的类型系统，从而可组合、可推断、可作为泛型实参传播。

```
%unique T   →   Unique<T>
%shared T   →   Shared<T>
%weak T     →   Weak<T>
%borrow T   →   Borrow<T>      (受 using 作用域约束的临时借用)
%loan T     →   Loan<T>        (可变借用 / 独占借出)
```

这些是编译器**内建（intrinsic）泛型**：用户不能重新定义，但它们走和用户泛型完全相同的闭型实例化与约束检查管线（见 [00-current-state.md](00-current-state.md) §3）。运行时不引入新结构——`Unique<T>` 闭型最终仍映射到既有的 `EZrOwnershipValueKind::UNIQUE` 与 `SZrOwnershipControl`（[ownership.h](../../../zr_vm_core/include/zr_vm_core/ownership.h)）。**这是纯前端表面重写 + 类型系统整合，运行时所有权机制不动。**

## 2. 为什么泛型化是对的

1. **可传播**：`Array<Unique<Texture>>`、`Map<string, Shared<Session>>` 有了统一表达。今天 `ownershipQualifier` 挂在单个 `SZrType` 上，嵌套进容器实参时无处安放。
2. **可被泛型代码统一处理**：`fn release<T>(x: Unique<T>)` 能对任意 owner 写一次。
3. **`%` 语义收敛**：`%` 长期只留给编译器指令/保留字（`%module/%import/%test/%async/%type/%detach`）。所有权种类不再占用 `%` 命名空间。
4. **约束可表达**：用 `where` 子句表达"这个泛型参数必须是 owner 类型"。

## 3. 类型形态与转换（状态机）

所有权的合法转换是一台状态机，泛型化后这台状态机由内建方法/intrinsic 表达，签名即规则：

```
Unique<T> ──share──▶ Shared<T> ──weak──▶ Weak<T>
   │                    ▲                   │
   │                    └──── upgrade ──────┘   (Weak<T> -> Shared<T>?，运行期可能失败 -> 返回 union)
   │
   ├── borrow ─▶ Borrow<T>     (只读借用，受 using 词法作用域约束，不转移所有权)
   ├── loan ───▶ Loan<T>       (独占可变借出，借出期间原 Unique 不可访问)
   ├── detach ─▶ T  (plain GC)  (显式脱离 owner 世界，对应 DetachValue)
   └── (作用域退出) ─▶ drop      (确定性析构，对应 ReleaseValue)
```

映射到既有运行时 API（[ownership.h](../../../zr_vm_core/include/zr_vm_core/ownership.h)，不改签名）：

| 泛型表面                          | 内建操作                             | 运行时函数                           |
| --------------------------------- | ------------------------------------ | ------------------------------------ |
| `Unique<T>(obj)` 构造           | `ZR_OWNERSHIP_BUILTIN_KIND_UNIQUE` | `ZrCore_Ownership_InitUniqueValue` |
| `x.share()` → `Shared<T>`    | `...KIND_SHARED`                   | `ZrCore_Ownership_ShareValue`      |
| `s.weak()` → `Weak<T>`       | `...KIND_WEAK`                     | `ZrCore_Ownership_WeakValue`       |
| `w.upgrade()` → `Shared<T>?` | `...KIND_UPGRADE`                  | `ZrCore_Ownership_UpgradeValue`    |
| `x.borrow()` → `Borrow<T>`   | `...KIND_BORROW`                   | `ZrCore_Ownership_BorrowValue`     |
| `x.loan()` → `Loan<T>`       | `...KIND_LOAN`                     | `ZrCore_Ownership_LoanValue`       |
| `x.detach()` → `T`           | `...KIND_DETACH`                   | `ZrCore_Ownership_DetachValue`     |
| 作用域 drop                       | `...KIND_RELEASE`                  | `ZrCore_Ownership_ReleaseValue`    |
| loan 作用域退出                   | `...KIND_RETURN_LOAN`              | `ZrCore_Ownership_ReturnLoanValue` |

> 注意 `upgrade` 返回 `Shared<T>?`（可空 / union）——弱引用升级可能因对象已析构失败。这是 union 类型（[04-union-types.md](04-union-types.md)）的第一个内建用例。

## 4. `using` 作为 drop 作用域

owner 值的析构时机绑定到 `using` 块的词法边界。把所有权绑定与作用域统一：

```zr
// 形式 A：using 直接引入并拥有一个 owner，块退出时确定性 drop
using (var tex = Unique<Texture>(loadTexture("a.png"))) {
    tex.bind();
    // ... 块结束自动 ReleaseValue(tex)，不进 GC
}

// 形式 B：借用作用域 —— Borrow<T> 的生命周期 = using 块
using (var view = tex.borrow()) {
    sampler.read(view);   // view: Borrow<Texture>，只读
    // 块退出借用结束，tex 重新可独占访问
}

// 形式 C：独占可变借出
using (var [slot] = buffer.loan()) {
    slot.write(0, 42);    // 借出期间 buffer 本身不可访问（编译期借用检查）
}
```

编译落点：复用 `compile_using_statement()`（[compile_statement.c:2197](../../../zr_vm_parser/src/zr_vm_parser/compiler/compile_statement.c#L2197)）已有的 `SZrDeterministicCleanupStep` 机制。差异是 cleanup step 的 `kind` / 触发的 builtin 由资源的**所有权泛型实参**决定（`Unique`→release，`Borrow`→end-borrow，`Loan`→return-loan），而非今天写死的 `callsClose + callsDestructor`。

## 5. `where` 约束扩展

新增内建约束谓词，让泛型代码能要求所有权种类：

```zr
func consume<T>(x: Unique<T>) where T: class { ... }   // 接管所有权

func observe<T>(w: Weak<T>): Shared<T>?                  // 仅弱观察
where T: owner                                        // 新增：T 必须是 owner 世界类型
{
    return w.upgrade();
}
```

约束谓词建议（落在 `SZrParameter` 现有的 `genericRequires*` 旁边，新增标志位）：

- `owner` —— 必须是 owner 世界类型（非 plain GC）。
- `unique` / `shared` / `weak` —— 要求特定种类（少用，主要给库作者）。

## 6. 语法糖与过渡

过渡期保留 `%unique T` 等作为 `Unique<T>` 的**纯语法糖**，parser 在 `try_get_ownership_qualifier()` 处把限定符直接归一化为内建泛型闭型，AST 下游只见 `SZrGenericType`：

```
%unique T   ⟶  desugar  ⟶  Unique<T>
```

这样：

- 已有源码与测试无需立即重写；
- 类型系统、metadata、AOT 后端只需认识内建泛型一条路径；
- 旧 `%unique T` 等类型语法现在会产生非阻断 `legacy_ownership_type_syntax` warning，提示迁移到 `Unique<T>` 等泛型写法，最终再移除 `ownershipQualifier` 字段。

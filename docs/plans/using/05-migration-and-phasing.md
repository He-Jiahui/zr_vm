# 05 · 分阶段落地、兼容与测试矩阵

四条主线（所有权泛型化 / `using` 守卫 / metadata token / union）彼此有依赖。本篇给出阶段切分、兼容策略与验收基线。

## 1. 依赖关系

```
[P0 metadata token]  ──┐
        │              ├──▶ [P2 using 插件守卫]  (需要 token 做签名比对)
[P1 所有权泛型化] ─────┘
        │
        └──▶ [P3 union]  (upgrade / PluginLoad / Option 复用 union)
                  │
                  └──▶ [P2 union 解构守卫]  (using 角色 D)
```

建议顺序：**P1 → P3 → P0 → P2**。理由：P1 是纯前端表面重写、风险最低且立刻收敛 `%`；P3（union）独立且为 P2 提供 `PluginLoad`/`Option` 表达；P0（token）是 P2 的前置；P2 最后整合。

# Semantic Stage 1 Semantic Query API

## Scope
- Add the Stage 1 `semantic_query.h` foundation from `docs/plans/lsp/01-semantic-inference-core.md`.
- Affected layers: parser public API, semantic query implementation, compiler frontend semantic diagnostics publication, parser semantic test registration, LSP semantic query consumption, and semantic documentation.

## Baseline
- `SZrSemanticContext` already stored expression, reference, numeric, reachability, logical, and ownership facts.
- Callers had to know the individual fact APIs directly; there was no shared parser-level query facade for LSP/debug/REPL.
- The initial `Diagnostics` query returned only a valid empty list; CFG reachability facts were not yet exposed as structured query diagnostics.
- Before the compiler publication slice, `compile_script` could emit reachability facts during type inference, but it never queried `ZrParser_SemanticQuery_Diagnostics`; `SZrSemanticContext.queryDiagnostics` therefore stayed empty after successful compilation.

- `tests/parser/test_numeric_branch_refinement.c`
  - True-branch simple integer comparison refinement narrows a registered identifier range in a temporary branch scope: `seed` registered as `0..255` in `if (seed < 10) { seed + 1; }` produces branch expression range `1..10`.
  - True-branch equality refinement intersects the registered identifier range with a singleton: `if (seed == 10) { seed + 1; }` produces branch expression range `11..11`.
  - True-branch edge-inequality refinement applies a single-interval complement only when the excluded point is an interval edge: `if (seed != 0) { seed + 1; }` produces branch expression range `2..256`.
  - True-branch simple logical-AND refinement applies both comparison bounds in one temporary branch scope chain: `seed` registered as `0..255` in `if (seed > 2 && seed < 10) { seed + 1; }` produces branch expression range `4..10`.
  - False-branch simple integer comparison refinement narrows the registered identifier range by inverting the comparison bound in a temporary branch scope: `seed` registered as `0..255` in `if (seed < 10) { seed + 1; } else { seed + 1; }` produces else-branch expression range `11..256`.
  - False-branch edge-equality refinement applies a single-interval complement only when the excluded equality point is an interval edge: `if (seed == 0) { ... } else { seed + 1; }` produces else-branch expression range `2..256`.
  - False-branch inequality refinement inverts `!=` to an exact equality singleton: `if (seed != 10) { ... } else { seed + 1; }` produces else-branch expression range `11..11`.
  - False-branch simple logical-OR refinement inverts both comparison bounds and intersects them in one temporary branch scope chain: `seed` registered as `0..255` in `if (seed <= 2 || seed >= 10) { seed + 100; } else { seed + 1; }` produces else-branch expression range `4..10`.
  - False-branch logical-AND same-direction complement refinement merges touching or overlapping inverted ranges for the same identifier: `if (seed < 10 && seed < 20) { ... } else { seed + 1; }` produces else-branch expression range `11..256`.
  - True-branch logical-OR same-direction refinement merges touching or overlapping true ranges for the same identifier: `if (seed < 10 || seed < 20) { seed + 1; }` produces then-branch expression range `1..20`.
  - Unary logical-not branch polarity refinement delegates supported conditions to the opposite branch scope: `if (!(seed < 10)) { seed + 1; }` produces then-branch expression range `11..256`.
  - True-branch logical-OR disjoint segment refinement preserves a fixed two-segment payload: `seed` registered as `0..255` in `if (seed < 10 || seed > 20) { seed + 1; }` produces envelope `1..256` with segments `1..10` and `22..256`.
  - True-branch nested logical-OR/AND segment refinement first reduces same-identifier child conditions, then preserves a fixed two-segment payload: `if ((seed > 2 && seed < 10) || seed == 20) { seed + 1; }` produces envelope `4..21` with segments `4..10` and `21..21`.
  - Else-if scope stacking preserves the outer false branch and inner true branch ranges: after `if (seed < 10) ... else if (seed < 20) { seed + 1; }`, the inner branch expression range is `11..20`.
  - False-branch logical-AND disjoint segment coverage preserves the fixed two-segment complement payload: `seed` registered as `0..255` in `if (seed > 10 && seed < 20) { ... } else { seed + 1; }` produces envelope `1..256` with segments `1..11` and `21..256`.

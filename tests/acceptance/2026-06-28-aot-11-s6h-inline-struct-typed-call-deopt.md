# 2026-06-28 AOT 11-S6H Inline-Struct Typed Call Metadata Guard/Deopt

Status: completed support sub-slice for 11-S6. Full 11-S6 remains open for cross-module token resolve integration and broader no-crash ABI drift injection.

## Scope

- Extend the typed direct-call metadata guard/deopt path to inline-struct `CALL_TYPED` generated by value SemIR lowering.
- Cover ordinary inline-struct direct calls, shared generic METHOD-slot calls, and full-AOT shared generic calls whose callee index is statically collected.
- Keep compatible direct `ZrLibrary_AotRuntime_CallInlineStruct()` paths unchanged.
- On caller/callee binding drift, fall back through `ZrLibrary_AotRuntime_CallInlineStructDynamicDeoptBridge()` so the interpreter path can copy the inline-struct return bytes into the generated destination slot.

## Baseline

- 11-S6D through 11-S6G covered scalar i64/u64/f64/bool direct-call guard/deopt.
- Inline-struct `CALL_TYPED` still emitted unconditional `CallInlineStruct()` calls in `backend_aot_c_value_semir_calls.c`.
- The existing full-AOT generic call test proved missing generic METHOD-slot deopt was omitted, but its bridge assertion was broader than that semantic and blocked metadata-drift deopt coverage.

## RED

- `tests/parser/test_aot_c_value_semir_contracts.c` now requires `zr_aot_value_exec_call_typed_metadata_guard`, `ZrLibrary_AotRuntime_CanUseTypedDirectCall(state, &frame, %u)`, `ZrLibrary_AotRuntime_CallInlineStructDynamicDeoptBridge(state,`, and the drift label.
- WSL gcc RED failed `zr_vm_aot_c_value_semir_contracts_test`: `4 Tests 1 Failures 0 Ignored`.
- Failure detail: missing source contract text `zr_aot_value_exec_call_typed_metadata_guard`.

## GREEN

- `backend_aot_c_value_semir_calls.c` now emits a reusable inline-struct typed-call metadata guard before direct `CallInlineStruct()` calls.
- Compatible caller/callee bindings still invoke the direct thunk.
- Binding drift emits `zr_aot_value_exec_call_typed_metadata_deopt` and routes through `CallInlineStructDynamicDeoptBridge()` with destination slot, argument base slot/count, destination layout id, byte offset/size, and deopt id.
- Shared generic METHOD-slot output still keeps the missing-instance deopt branch; full-AOT shared generic output still omits the missing-instance branch while allowing the metadata-drift bridge.
- `tests/parser/test_aot_c_generic_call_typed.c` now narrows the full-AOT assertion to forbid only missing-instance deopt and require the metadata guard/deopt surface.

## Acceptance Decision

Accepted for 11-S6H inline-struct `CALL_TYPED` metadata guard/deopt scope. Inline-struct typed direct calls now check caller/callee metadata binding compatibility before direct thunk invocation and deopt through the inline-struct dynamic bridge on drift. Cross-module token resolve integration and broader no-crash ABI drift injection remain open 11-S6 work.
